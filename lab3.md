# Лабораторная работа №3 - Бизнес-процессы предметной области (AS-IS)

---

## Основной процесс: Выполнение проекта по договору

### Паспорт процесса

| Параметр | Значение |
|----------|----------|
| **Название** | Выполнение проекта по договору |
| **Цель** | Реализовать проектные работы в срок, с требуемым качеством, с учётом доступных ресурсов (люди, оборудование) и зафиксировать финансовый результат |
| **Инициатор** | Заказчик (подписание договора) / Руководство организации (запуск проекта) |
| **Входные данные** | • Договор с техническим заданием<br>• Требования заказчика<br>• Список доступных сотрудников по категориям (конструкторы, инженеры, лаборанты)<br>• Реестр оборудования (отделское / общее)<br>• Календарный план работ |
| **Выходные данные** | • Готовая проектная документация<br>• Акт сдачи-приёмки работ<br>• Отчёт о фактической стоимости проекта<br>• Обновлённые данные: загрузка сотрудников, статус оборудования, выполнение договора |
| **Участники** | • Руководитель проекта / договора<br>• Начальники отделов<br>• Конструкторы, инженеры, техники, лаборанты<br>• Бухгалтерия (стоимостной учёт)<br>• Субподрядные организации (при привлечении) |
| **Ожидаемый результат** | Проект сдан, договор закрыт, оплата получена, все данные учтены в системе для последующих запросов |



## Вспомогательные процессы

### Процесс №1: Распределение сотрудников по проектам

| Параметр | Значение |
|----------|----------|
| **Цель** | Сформировать проектную группу из сотрудников нужных категорий с учётом их квалификации, загрузки и принадлежности к отделам |
| **Инициатор** | Руководитель проекта |
| **Входные данные** | • Заявка на проект (требуемые роли, сроки)<br>• Список сотрудников с атрибутами: категория, отдел, авторские свидетельства (для конструкторов), текущая загрузка |
| **Выходные данные** | • Утверждённый состав проектной группы<br>• Обновлённые данные о загрузке сотрудников |
| **Участники** | Руководитель проекта, начальники отделов |
| **Ожидаемый результат** | Проект укомплектован кадрами, назначения зафиксированы, нет конфликтов по занятости |

### Процесс №2: Учёт и выдача оборудования под проект

| Параметр | Значение                                                                                                                                                    |
|----------|-------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Цель** | Обеспечить проект необходимым оборудованием, избегая конфликтов использования между проектами и отделами                                                    |
| **Инициатор** | Руководитель проекта                                                                                                                                        |
| **Входные данные** | • Заявка на оборудование (тип, срок)<br>• Реестр оборудования: привязка к отделам / общее пользование<br>• График занятости оборудования по другим проектам |
| **Выходные данные** | • Маршрутный лист передачи оборудования<br>• Обновлённый статус: «закреплено за проектом Х»<br>• Акт приёма-передачи                                        |
| **Участники** | Руководитель проекта, начальники отделов                                                                                                                    |
| **Ожидаемый результат** | Оборудование выдано под проект, учтено в системе, вовремя возвращено, нет простоев из-за нехватки                                                           |
