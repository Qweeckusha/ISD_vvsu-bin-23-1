# Анализ предметной области: ИС проектной организации

## Цель и задачи проекта

### Основная цель  
Создание единой информационной системы для автоматизации учёта кадров, договоров, проектов, оборудования и выполненных работ в проектной организации.

### Основные задачи  
- Обеспечить централизованный учёт сотрудников с разбивкой по категориям и отделам.  
- Вести учёт договоров и проектов, включая их взаимосвязи и ответственных лиц.  
- Осуществлять распределение и отслеживание использования оборудования между проектами и отделами.  
- Фиксировать участие сотрудников и подрядных организаций в проектах и договорах.  
- Обеспечить стоимостной учёт выполненных работ и эффективность использования ресурсов.  
- Реализовать гибкие механизмы формирования отчётов и выборок по заданным критериям.

## Основные требования  

- Система должна поддерживать хранение и обработку данных о сотрудниках, договорах, проектах, оборудовании и субподрядчиках.  
- Должна быть обеспечена возможность многозначных связей между договорами и проектами.  
- Необходимо учитывать динамическое распределение оборудования между проектами и отделами.  
- Все данные должны быть доступны для анализа и формирования отчётов в соответствии с перечнем запросов из ТЗ.  
- Система должна поддерживать историю изменений (например, передачу оборудования или участие в проектах на определённую дату).

## Основная функциональность проекта  

- Регистрация и редактирование информации о сотрудниках, отделах и должностях.  
- Учёт договоров и проектов с указанием руководителей и состава исполнителей.  
- Учёт оборудования с возможностью привязки к отделам или проектам.  
- Учёт участия субподрядных организаций в проектах.  
- Расчёт стоимости договоров на основе выполненных работ.  
- Формирование стандартных отчётов по заданным шаблонам (см. список запросов).

## Дополнительные требования и ограничения  

- Интерфейс - простой и интуитивно понятный (в первую очередь для внутренних пользователей - сотрудников организации).  
- Платформа - веб-приложение (доступ через браузер), без привязки к конкретной ОС.  
- Архитектура - клиент-серверная, с возможностью масштабирования.  
- Безопасность - авторизация и разграничение прав доступа (например, начальник отдела видит только свой отдел, администратор - всю систему).  
- Поддержка резервного копирования и восстановления данных.

## Целевая аудитория проекта  

- Руководство проектной организации (для аналитики и контроля).  
- Начальники отделов (для управления кадрами и ресурсами).  
- Бухгалтерия и планово-экономический отдел (для стоимостного учёта).  
- Администраторы ИС (для настройки и поддержки системы).

# Основные сущности и их атрибуты

| Сущность                     | Атрибуты                                                                                                                                        |
|-----------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------|
| **Сотрудник**               | `id`, `ФИО`, `дата_рождения`, `должность`, `категория` (конструктор/инженер/техник/лаборант/обслуживающий персонал), `id_отдела`                |
| **Конструктор** *(подтип)*  | `число_авторских_свидетельств`                                                                                                                  |
| **Инженер** *(подтип)*      | - *(может не иметь уникальных атрибутов, но может быть руководителем договора/проекта)*                                                         |
| **Техник** *(подтип)*       | `обслуживаемое_оборудование` (список типов или моделей оборудования)                                                                            |
| **Лаборант / Обсл. персонал** | - *(могут не иметь уникальных атрибутов в рамках задания)*                                                                                      |
| **Отдел**                   | `id`, `название`, `id_начальника`                                                                                                               |
| **Договор**                 | `номер`, `заказчик`, `дата_заключения`, `срок_выполнения`, `id_руководителя`, `статус` (текущий/завершён)                                       |
| **Проект**                  | `id`, `название`, `описание`, `дата_начала`, `дата_окончания`, `id_руководителя`, `статус`                                                      |
| **Оборудование**            | `инвентарный_номер`, `тип/модель`, `принадлежность` (`отдел` / `общее`), `id_текущего_владельца` (отдел или проект), `статус` (свободно/занято) |
| **Субподрядная организация**| `id`, `название`, `контактные_данные`                                                                                                           |
| **Участие сотрудника**      | `id_сотрудника`, `id_договора` или `id_проекта`, `роль`, `дата_начала`, `дата_окончания`                                                        |
| **Использование оборудования** | `id_оборудования`, `id_проекта`, `дата_начала`, `дата_окончания`                                                                                |
| **Работы субподрядчика**    | `id_субподрядчика`, `id_проекта`, `описание_работ`, `стоимость`, `дата_выполнения`                                                              |

> **Примечания:**
> - Руководителем договора или проекта может быть **только инженер или конструктор**.
> - Оборудование из отдела может быть передано другому отделу или проекту **только если оно свободно**.
> - Один договор может включать несколько проектов, и один проект - участвовать в нескольких договорах (многие-ко-многим).
